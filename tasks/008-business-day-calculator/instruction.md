# タスク: 日本営業日カレンダーCLI

Pythonで、日本の営業日を扱うCLIツールを作成してください。

## ゴール

土日・日本の祝日・振替休日・国民の休日・年末年始休業・会社独自休業日を考慮して、営業日判定や加算・差分計算ができるツールを実装してください。

## コマンド

```bash
python bizday.py is-bizday 2026-05-07
python bizday.py add 2026-04-30 3
python bizday.py diff 2026-04-30 2026-05-08
python bizday.py list-month 2026-05
```

オプション:

- `--company-holidays <json>`: 会社独自休業日を追加
- `--inclusive`: `add` で開始日が営業日なら0日目として数える
- `--inclusive-end`: `diff` で終了日を含める
- `--no-year-end`: 年末年始休業（12/29〜1/3）を無効化する
- `--format json`: JSONで出力する

## 対応範囲

- 2019年〜2030年の日付を正しく扱うこと
- タイムゾーンは `Asia/Tokyo` 前提で、**日付のみ**を扱えばよい
- 日本の祝日は以下を考慮すること
  - 固定祝日
  - ハッピーマンデー対象祝日
  - 春分日・秋分日（表や定数で実装してよい）
  - 振替休日
  - 国民の休日
  - 2019年・2020年・2021年の特例休日移動も扱うこと

## 営業日定義

営業日ではない日:

1. 土曜日
2. 日曜日
3. 日本の祝日
4. 振替休日
5. 国民の休日
6. 年末年始休業（デフォルトで `12/29`〜`1/3`）
7. `--company-holidays` で追加された休業日

## company-holidays の形式

次のいずれかを含むJSON配列を受け付けること。

```json
[
  "2026-05-01",
  {"start": "2026-08-13", "end": "2026-08-16"}
]
```

- 文字列は単日休業
- オブジェクトは両端を含む休業期間

## 各サブコマンド仕様

### is-bizday

- 指定日が営業日なら `true`、そうでなければ `false` を返す
- テキスト出力では理由も表示すること

例:

```text
2026-05-05: false (こどもの日)
```

### add

- 指定日から `n` 営業日後の日付を返す
- `n` は負数も可（過去方向へ移動）
- デフォルトでは開始日はカウントしない
- `--inclusive` 指定時のみ、開始日が営業日なら `0日目` として扱う

例:

```text
$ python bizday.py add 2026-04-30 3
2026-05-08
```

### diff

- 2つの日付の間に存在する営業日数を返す
- デフォルトでは `[start, end)` とし、終了日は含めない
- `--inclusive-end` 指定時は `[start, end]` とする
- `start > end` の場合は負の値を返してよい

### list-month

- 指定月の各日について、営業日かどうかと理由を一覧表示すること
- `--format json` では配列で返すこと

## 実装要件

- 祝日判定ロジックを関数として分離すること
- 年度や月をまたぐ計算でも正しく動くこと
- `pytest` で実行できるテストを作成すること

## テストで必ずカバーすること

- 通常の土日判定
- 振替休日
- 国民の休日
- 2019年の即位関連休日
- 2020年・2021年の祝日移動
- ゴールデンウィーク付近の `add` / `diff`
- 年末年始休業あり/なし
- 会社独自休業日の追加
- 負方向の `add` / `diff`

## 制約

- Python標準ライブラリのみ使用すること
- 春分日・秋分日は 2019〜2030 の表を埋め込んでもよい
- `python bizday.py ...` で実行できるようにすること
