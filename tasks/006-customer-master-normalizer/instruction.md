# タスク: 顧客マスタ正規化CLI

Pythonで、日本の顧客マスタCSVを正規化するCLIツールを作成してください。

## ゴール

顧客データにありがちな表記ゆれや入力崩れを吸収し、**正規化済みCSV** と **エラーレポートJSON** を生成するツールを実装してください。

## コマンド

```bash
python customer_clean.py --input raw.csv --output cleaned.csv --errors errors.json
```

オプション:

- `--keep-invalid`: バリデーションエラーがある行も `cleaned.csv` に残す
- `--encoding <encoding>`: 入力CSVのエンコーディング（省略時は `utf-8-sig` で読み、失敗したら `utf-8`）

## 入力CSVの列

次のヘッダを持つCSVを想定してください。

```csv
customer_id,name,name_kana,postal_code,prefecture,city,address_line,phone,email
```

## 出力CSV

出力CSVは次の列を持つこと。

```csv
customer_id,name,name_kana,postal_code,prefecture,city,address_line,normalized_address,phone,email,is_valid
```

- `normalized_address` は `prefecture + city + address_line` を連結したもの
- `is_valid` は `true` / `false`

## 正規化仕様

### 共通

- 前後空白を除去すること
- 連続する空白は1つにまとめること
- 全角数字は半角数字に揃えること
- 全角英字は半角英字に揃えること

### name

- 先頭末尾の空白除去
- 連続空白の圧縮
- それ以外は過剰に変換しないこと

### name_kana

- ひらがなは全角カタカナに揃えること
- 半角カタカナは全角カタカナに揃えること
- 空白は除去すること
- 中点 `・` `･` は保持してよい

### postal_code

以下のいずれかを受け付け、最終的に `NNN-NNNN` 形式へ正規化すること。

- `1600023`
- `160-0023`
- `１６０−００２３`

7桁にならない場合はエラー。

### prefecture

- 正式な都道府県名はそのまま通す
- 次の別名は正式名へ変換すること
  - `東京` → `東京都`
  - `大阪` → `大阪府`
  - `京都` → `京都府`
  - `北海道` → `北海道`
  - `神奈川` → `神奈川県`
  - `愛知` → `愛知県`
  - `福岡` → `福岡県`
  - `沖縄` → `沖縄県`
- 上記以外の不正値はエラーにすること

### city / address_line

- 全角数字を半角数字に変換すること
- 前後空白を除去し、連続空白を1つにまとめること
- `address_line` では、以下のような **数字だけで構成される丁目・番・号** をハイフン区切りへ正規化すること
  - `西新宿2丁目8番1号` → `西新宿2-8-1`
  - `丸の内１丁目９番２号` → `丸の内1-9-2`
  - `1丁目2-3` → `1-2-3`
- 漢数字の丁目変換（例: `二丁目`）は対応不要

### phone

数字のみを抜き出してから、以下のルールで整形すること。

- `070` / `080` / `090` / `050` で始まる11桁: `3-4-4`
- `03` / `06` で始まる10桁: `2-4-4`
- 上記以外の10桁: `3-3-4`
- 条件に合わない場合はエラー

例:

- `09012345678` → `090-1234-5678`
- `03-1234-5678` → `03-1234-5678`
- `0751234567` → `075-123-4567`

### email

- 前後空白除去
- 英字は小文字化してよい
- 簡易的なメール形式チェックを行うこと
  - 例: `local@domain` を満たさない場合はエラー

## バリデーション

1行ごとに複数エラーがあり得ること。

エラーJSONの形式は以下のようにすること。

```json
[
  {
    "line": 3,
    "customer_id": "C002",
    "errors": [
      "postal_code: 7桁ではありません",
      "email: 不正な形式です"
    ]
  }
]
```

- `line` はヘッダを1行目としたCSV上の行番号
- `customer_id` が空の場合でも落ちないこと

## 実装要件

- 正規化ロジックとCLIを分離すること
- `pytest` で実行できるテストを作成すること
- テスト用のfixture CSVを用意してよい
- 不正行が1件もなくても `errors.json` は空配列 `[]` を出力すること

## テストで必ずカバーすること

- 郵便番号の全角/半角/ハイフン有無
- 都道府県別名の正規化
- `name_kana` のひらがな/半角カナ → 全角カナ
- `address_line` の `丁目/番/号` 正規化
- 電話番号の 11桁携帯、東京/大阪の市外局番、その他10桁
- 複数エラーが同時に出るケース
- `--keep-invalid` の動作

## 制約

- Python標準ライブラリのみ使用すること
- `python customer_clean.py ...` または同等の分かりやすい実行方法にすること
