# タスク: 日本語ゆらぎ検索CLI

TypeScriptで、日本語の表記ゆれを吸収して検索できるCLIツールを作成してください。

## ゴール

顧客名・会社名・カナ表記などに対して、**全角/半角**、**ひらがな/カタカナ**、**スペースや中点の有無**、**英字の大文字小文字**の違いを吸収して検索できるツールを実装してください。

コマンド名は `jpfind` とします。

## 使い方

```bash
jpfind --data <JSONファイル> --query "<検索語>"
jpfind --data customers.json --query "ｻﾄｳ ﾊﾅｺ"
jpfind --data customers.json --query "株式会社 でんきー" --fields company,name
jpfind --data customers.json --query "SATO" --strict --format json
```

## 入力データ

- JSONファイルは **オブジェクト配列** とすること
- 各要素は少なくとも `id` を持つこと
- 文字列フィールドと数値フィールドが混在してよい

例:

```json
[
  {
    "id": 1,
    "name": "佐藤 花子",
    "name_kana": "ｻﾄｳﾊﾅｺ",
    "company": "株式会社デンキー",
    "email": "hanako@example.com"
  },
  {
    "id": 2,
    "name": "さとうはなこ",
    "name_kana": "サトウ ハナコ",
    "company": "（株）でんきー",
    "email": "sato@example.com"
  }
]
```

## 検索仕様

### 基本動作

- デフォルトでは、すべての**文字列フィールド**を検索対象にすること
- `--fields name,name_kana,company` のように指定された場合、そのフィールドのみを検索対象にすること
- クエリ中の空白でトークン分割し、**AND検索**にすること
  - 例: `"ｻﾄｳ ﾊﾅｺ"` は、正規化後に `サトウ` と `ハナコ` の両方を含むレコードのみ一致
- 一致したレコードを表示すること

### 通常モードの正規化ルール

通常モード（`--strict` なし）では、検索時に以下を行うこと。

1. Unicode正規化（NFKC）
2. 英字は大文字小文字を区別しない
3. ASCIIスペースと全角スペースは無視する
4. 中点 `・` `･` は無視する
5. ひらがなはカタカナに寄せて比較する
6. 半角カタカナは全角カタカナに寄せて比較する
7. 次のハイフン類は無視する: `-`, `‐`, `‑`, `―`, `−`
8. 半角長音 `ｰ` は全角長音 `ー` として扱う

### strictモード

`--strict` 指定時は、以下のみ行うこと。

- Unicode正規化（NFKC）
- 英字の大文字小文字の吸収

つまり、**かな変換・スペース除去・中点除去・ハイフン無視は行わない**こと。

## ソート順

一致結果は次の優先順位で並べること。

1. いずれかの対象フィールドが、正規化後クエリ全体と**完全一致**
2. いずれかの対象フィールドが、正規化後クエリ全体で**前方一致**
3. その他の部分一致
4. 同順位なら `id` の昇順

## 出力形式

### 標準出力（デフォルト）

表形式で以下の列を出力すること。

- `id`
- `score`（`exact` / `prefix` / `partial`）
- 一致に使われた主なフィールド名
- レコードのJSON要約（1行で見やすく整形）

出力例:

```text
id  score   field        record
1   exact   name_kana    {"id":1,"name":"佐藤 花子","company":"株式会社デンキー"}
2   partial company      {"id":2,"name":"さとうはなこ","company":"（株）でんきー"}
```

### JSON出力

`--format json` 指定時は、以下の配列をJSONで出力すること。

```json
[
  {
    "id": 1,
    "score": "exact",
    "matched_fields": ["name_kana"],
    "record": {"id": 1, "name": "佐藤 花子", "company": "株式会社デンキー"}
  }
]
```

## 実装要件

- 検索ロジックをライブラリとして再利用できるようにすること
  - 例: `normalizeJapaneseText()`, `searchRecords()` のような関数
- CLIからそのライブラリを呼び出す構成にすること
- 不正なJSONや存在しないフィールド指定に対して、分かりやすいエラーメッセージを出すこと
- レコード件数が0件でもクラッシュしないこと
- `npm test` で実行できるテストを作成すること

## テストで必ずカバーすること

- 全角/半角英数字
- ひらがな/カタカナ/半角カタカナ
- スペース・全角スペース・中点の有無
- AND検索
- strictモードと通常モードの差
- ソート順（exact / prefix / partial）
- 存在しないフィールド指定時のエラー

## 制約

- 外部の全文検索エンジンや検索専用ライブラリは使わないこと
- Unicode正規化や基本的な文字変換に標準機能を使うのは可
- `npm script` または `npx` で実行できるようにすること
