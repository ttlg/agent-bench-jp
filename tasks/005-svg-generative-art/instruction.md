# タスク: SVGジェネレーティブアートエンジン

単一のHTMLファイル (`index.html`) で、インタラクティブなジェネレーティブアートを生成するエンジンを実装してください。
外部ライブラリ・CDN一切禁止。純粋なHTML + SVG + CSS + JavaScriptのみ。

## 要求機能

### 1. アートモード（最低3種類）

以下のうち**3つ以上**を実装すること：

- **パーティクルフロー**: 数百のパーティクルがフローフィールド（Perlinノイズ的なベクトル場）に沿って流れる。パーティクルは軌跡を残し、徐々にフェードアウトする
- **フラクタルツリー**: 再帰的に枝分かれする樹木。風に揺れるアニメーション付き。深さ・角度・枝の太さをパラメータ化
- **波動干渉**: 複数の波源から同心円状の波が広がり、干渉パターンを生成。波源はクリックで追加可能
- **万華鏡**: N回対称の万華鏡パターン。マウス位置で模様がリアルタイムに変化
- **螺旋銀河**: フィボナッチ螺旋に基づく星の配置。回転アニメーション。星の色はスペクトル分布

### 2. インタラクション

- **マウス/タッチ**: ポインター位置がアートに影響を与える（引力、波源追加、パターン変形など）
- **クリック**: モード固有のアクション（波源追加、パーティクルバースト、枝の再生成など）

### 3. コントロールパネル

SVGの外側に操作UIを配置：

- モード切替ボタン（各アートモードを選択）
- パラメータ調整スライダー（最低3つ。例: 速度、密度、色相）
- 再生/一時停止ボタン
- リセットボタン（現在のモードを初期状態に戻す）
- カラーテーマ切替（最低3テーマ。例: ネオン、パステル、モノクロ）

### 4. アニメーション

- `requestAnimationFrame` によるスムーズなアニメーション（60fps目標）
- SVG要素の動的生成・更新・削除を適切に管理
- パフォーマンスを意識し、不要な要素は削除またはリサイクルすること

### 5. ビジュアル品質

- SVGの特性を活かした、滑らかで美しい描画
- グラデーション、透過、ブレンドモードの活用
- 配色の調和（各テーマ内で統一感のある色使い）
- 背景は暗色系で、アート要素が映えるデザイン

### 6. レスポンシブ

- ウィンドウサイズに合わせてSVGが自動リサイズ
- viewBoxを適切に設定

## 技術仕様

- ファイル: `index.html`（単一ファイル。CSS/JSはインライン）
- キャンバス: SVG要素（`<canvas>` は使用禁止）
- 描画: SVGの `<circle>`, `<path>`, `<line>`, `<g>`, `<defs>` 等を活用
- アニメーション: JavaScriptで `requestAnimationFrame`。CSSアニメーションやSMILは補助的使用のみ可
- パフォーマンス: パーティクル系は最低200個以上を同時描画できること

## 評価のポイント

このタスクは通常のルーブリック（正確性・コード品質・堅牢性・設計・指示理解）に加え、**ビジュアル品質**も評価対象です：

- 数学的な正確さ（フラクタル、ノイズ、波動の物理シミュレーション）
- アニメーションの滑らかさと視覚的な美しさ
- インタラクションの応答性と自然さ
- 全体的なデザインの完成度（UIを含む）
