# 評価ルーブリック

5観点 x 5段階 = 25点満点

## 1. 正確性（Correctness）

要求を正しく満たしているか。

| スコア | 基準 |
|---|---|
| 5 | 全要求を完全に満たし、エッジケースも対応 |
| 4 | 主要要求を全て満たす |
| 3 | 大部分は正しいが、一部抜けがある |
| 2 | 基本的な部分にミスがある |
| 1 | 要求と大きく乖離 |

## 2. コード品質（Code Quality）

可読性、構造、命名、慣用的な書き方。

| スコア | 基準 |
|---|---|
| 5 | プロダクションにそのまま入れられる品質 |
| 4 | 軽微な改善点はあるが十分な品質 |
| 3 | 動くが、構造や命名に改善の余地 |
| 2 | 読みにくい、冗長、非慣用的 |
| 1 | 保守不能なレベル |

## 3. 堅牢性（Robustness）

エラーハンドリング、入力検証、エッジケース。

| スコア | 基準 |
|---|---|
| 5 | 想定外の入力にも適切に対応 |
| 4 | 一般的なエラーケースをカバー |
| 3 | 正常系は安全だが異常系に弱い |
| 2 | 基本的なエラー処理が欠けている |
| 1 | 容易にクラッシュする |

## 4. 設計判断（Design Decision）

適切な抽象化、ライブラリ選択、アーキテクチャ。

| スコア | 基準 |
|---|---|
| 5 | 優れた設計判断、拡張性も考慮 |
| 4 | 適切な設計、妥当な選択 |
| 3 | 動くが、過剰設計または過少設計 |
| 2 | 不適切な設計判断が目立つ |
| 1 | 根本的に設計がおかしい |

## 5. 指示理解（Instruction Comprehension）

日本語の指示をどれだけ正確に理解したか。

| スコア | 基準 |
|---|---|
| 5 | 暗黙の意図まで汲み取っている |
| 4 | 明示された要求を正確に理解 |
| 3 | 概ね理解しているが、細部の誤解あり |
| 2 | 重要な要求を見落としている |
| 1 | 指示を根本的に誤解している |
